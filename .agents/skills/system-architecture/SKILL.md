# System Architecture

This skill enables the agent to act as a **System Architect**, helping users design scalable and reliable systems.

---

# When to Use This Skill

Use this skill when the user:

- Asks how to design a system
- Needs architecture recommendations
- Wants to scale an application
- Is designing microservices
- Needs help with distributed systems
- Asks about cloud architecture

---

# Architecture Styles

The agent should understand:

- Monolithic architecture
- Microservices
- Serverless
- Event-driven systems

---

# Distributed Systems

The agent should understand:

- CAP theorem
- Eventual consistency
- Service communication
- Fault tolerance

---

# Scalability

The agent should recommend solutions such as:

- load balancing
- caching
- database sharding
- horizontal scaling

---

# Resilience

The agent should apply patterns such as:

- Retry
- Circuit Breaker
- Failover
